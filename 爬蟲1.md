# NCHU 最新消息擷取程式規劃
## Summary
建立一支獨立的 Python 腳本，從 `https://www2.nchu.edu.tw/news/page/0/id/2` 起抓取「最新消息」第 1 到第 10 頁，整理出 `日期`、`標題`、`發布單位`、`連結網址` 四個欄位，最後輸出為 `Excel (.xlsx)` 檔。

## Implementation Changes
- 使用純 HTTP 方案：`requests` 抓頁面、`BeautifulSoup` 解析 HTML。
- 腳本預設抓 10 頁，頁數與起始 URL 寫成常數，方便之後調整。
- 分頁處理採「可配置偏移或頁碼規則」設計：
  - 先以清單頁 URL 模式組出第 1 到第 10 頁。
  - 若站台實際使用偏移量而非單純頁碼，將分頁步長抽成常數，避免把規則寫死在邏輯裡。
- 每頁解析清單中的每則公告，擷取：
  - 日期：清單頁顯示日期；若缺漏，再考慮進入詳情頁補抓。
  - 標題：清單中的公告標題文字。
  - 發布單位：清單頁或詳情頁中的單位欄位。
  - 連結網址：轉成完整絕對網址。
- 輸出為 Excel：
  - 產生單一工作表，例如 `最新消息`。
  - 欄位順序固定為 `日期`、`標題`、`發布單位`、`連結網址`。
  - 檔名預設可設為 `nchu_latest_news.xlsx`。
- 加入基本穩定性處理：
  - `User-Agent` header
  - 逾時設定
  - 失敗頁面錯誤訊息提示
  - 去重複，避免同一公告重覆寫入

## Public Interfaces
- 腳本以命令列直接執行，例如 `python scrape_nchu_news.py`。
- 先維持簡單介面，不強制加入 CLI 參數；若要保留擴充性，可在檔案頂部集中設定：
  - `BASE_URL`
  - `TOTAL_PAGES = 10`
  - `OUTPUT_FILE = "nchu_latest_news.xlsx"`
  - `PAGE_STEP` 或等效分頁規則常數

## Test Plan
- 執行腳本後成功產生 `.xlsx` 檔案。
- Excel 首列欄名正確，且欄位順序符合需求。
- 至少抽查第 1 頁與第 10 頁各 1 到 2 筆資料，確認：
  - 日期格式合理
  - 標題非空
  - 發布單位非空或有明確 fallback
  - 連結可對到正確公告
- 驗證重跑腳本不會在同一輸出中產生重複資料。
- 驗證某一頁抓取失敗時，程式能提示錯誤而不是無訊息中斷。

## Assumptions
- 這次只需要單支可直接執行的 Python 程式，不需要包成模組或排程服務。
- 站台內容可透過純 HTTP 取得，不需要 Selenium / Playwright。
- 若清單頁無法完整提供 `發布單位`，實作會補抓公告詳情頁。
- 依賴套件可接受使用 `requests`、`beautifulsoup4`、`openpyxl`；若已有 `pandas` 也可用來輸出 Excel，但不是必要。
- 目前工作目錄沒有既有專案結構，因此預設新增單一腳本與必要的 `requirements.txt` 即可。
